# Start Here

This repository is a curated set of architecture patterns for designing data and AI systems.

It is organized into two parts:

- **foundations**: core architectural concepts that apply across systems  
- **applied-patterns**: examples of how those concepts are used in real-world designs  

---

## How to Read This Repository

If you are new to these concepts, start with the foundations:

1. canonical-identity  
2. semantic-layer  
3. retrieval-basics  
4. governance-lineage  

These provide the core building blocks used across all patterns.

---

## How to Navigate by Goal

**If you want to understand core architecture concepts**
- Read the entire `foundations` section in order

**If you want to see how systems are designed**
- Start with:
  - applied-patterns/internal-knowledge-assistant  
  - applied-patterns/document-ingestion-search  

**If you are working with limited resources**
- Start with:
  - applied-patterns/lightweight-ingestion  
  - applied-patterns/simple-metadata-catalog  

---

## What to Expect

These patterns are:

- architecture-first, not tool-specific  
- focused on structure, tradeoffs, and reasoning  
- intentionally simplified for clarity  

They are not full production implementations, but reflect how real systems are designed and decomposed.

---

## Suggested Approach

You do not need to read everything.

Start with one foundation concept and one applied pattern, then expand based on your needs.
